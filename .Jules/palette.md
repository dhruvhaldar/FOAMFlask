## 2024-05-24 - Added Explicit Focus States to Form Elements
**Learning:** Default browser focus states (or missing focus states on `outline-none` elements) severely hamper keyboard navigation visibility, especially in dense configuration forms. Adding explicit Tailwind focus rings (`focus:ring-2 focus:ring-cyan-500`) significantly improves the tab navigation experience. However, be cautious when universally applying focus utilities: elements visually hidden with `sr-only` (like custom file inputs) should not receive focus rings directly as they won't be visible in the expected location.
**Action:** When auditing forms for keyboard accessibility, systematically check for explicit `focus:` utility classes on all interactive elements (inputs, selects, buttons) while filtering out visually hidden inputs that act via proxy labels.
